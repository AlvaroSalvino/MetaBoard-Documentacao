Estrutura do Projeto MetaBoard

O projeto MetaBoard é estruturado da seguinte maneira:

Pasta Raiz

Na pasta raiz, estão dispostos os 4 principais arquivos:

  • app.py: Este é o arquivo principal que inicia a aplicação.
  • form.py: Este arquivo contém as definições dos formulários usados na aplicação.
  • views.py: Este arquivo define as rotas e as funções de visualização para a aplicação.
  • models.py: Este arquivo define os modelos de dados usados na aplicação.

Além desses arquivos, existem duas pastas principais: static e templates.

Pasta static

A pasta static é composta pelas pastas css e image.

  • A pasta css contém os arquivos de estilo CSS para a aplicação. Os arquivos incluem login.css, registro.css e style.css.
  • A pasta image contém os arquivos de imagem usados na aplicação.

Pasta templates

A pasta templates é composta por três pastas (evento, meta, models) e quatro arquivos HTML (area_de_trabalho.html, index.html, login.html, registro.html).

  • A pasta evento contém os arquivos HTML relacionados às tarefas (criar_evento.html, detalhar_evento.html, editar_evento.html, evento.html).
  • A pasta meta contém os arquivos HTML relacionados aos projetos (criar_meta.html, detalhar_meta.html, editar_meta.html, metas_fechadas.html, todas_as_metas.html).
  • A pasta models contém os arquivos HTML que definem a estrutura básica das páginas (base.html, model_head.html, model_page.html)
